# 貢献ガイド (CONTRIBUTING)

歓迎します！このリポジトリへ貢献する際の簡単なルールと、コミットメッセージのテンプレートを日本語でまとめます。

## 基本方針
- 変更は `dev` ブランチへ PR を作成してマージしてください。
- 小さな変更でも PR を作成してレビューを受けてください。

## コードスタイル
- 既存のスタイルに合わせてください。C# コードは既存の命名規則とインデントに従ってください。

## コミットメッセージ（日本語テンプレート）
一貫性のため、コミットメッセージは日本語で以下の形式を推奨します：

<種類>: <概要>

（空行）
詳細な説明（必要に応じて複数行）

- 種類は以下から選んでください：
  - `feat`：新機能の追加
  - `fix`：バグ修正
  - `docs`：ドキュメントのみの変更
  - `style`：フォーマットやコメント、コードの意味に影響しない変更
  - `refactor`：リファクタリング（機能追加やバグ修正を伴わない変更）
  - `perf`：パフォーマンス改善
  - `test`：テスト追加/修正
  - `chore`：ビルドプロセスや補助ツール、ライブラリ更新など

例：

fix: HotkeyManager のフック失敗時に RegisterHotKey へフォールバックするよう修正

HotkeyManager.InstallLowLevelHook でフックが失敗した場合に、
明示的なログ出力と RegisterHotKey でのフォールバックを行うようにしました。

また、Process.MainModule 取得時の例外を安全に処理します。

## プルリクエスト作成時のチェックリスト
- 変更点に対する簡単な説明を PR 本文に記載してください。
- 影響範囲が大きい変更は事前に Issue で相談してください。
- ビルドが成功することを確認してください：

```powershell
dotnet build Shuyu.sln -c Release
```

## 小さな注意
- 既にマージ済みのコミットメッセージを書き換える（履歴を書き換える）操作は慎重に行ってください。チームルールがある場合は従ってください。
